[role="lead"]
Amazon Web Services(AWS), Microsoft Azure 및 Google Cloud에 Cloud Volumes ONTAP을 배포할 수 있습니다.  `Quick create`를 사용하여 단일 노드 및 고가용성(HA) 구성 모두에서 더 빠른 배포가 가능합니다. 이 간소화된 프로세스는 단일 페이지에서 기본값을 자동으로 설정하고 탐색을 최소화하여 고급 방법에 비해 배포 단계를 줄여줍니다.

.단계
. 왼쪽 탐색 메뉴에서 *저장소 > 관리*를 선택합니다.
. [[구독]]캔버스 페이지에서 *시스템 추가*를 클릭하고 안내를 따르세요.
. 클라우드 공급자(*Amazon Web Services*, *Microsoft Azure*, *Google Cloud*)를 선택한 다음 *Cloud Volumes ONTAP/Cloud Volumes ONTAP HA* > *새로 추가*를 선택합니다. *빠른 생성* 옵션이 기본적으로 선택되어 있습니다.

.시스템 세부 정보
. *클라우드 공급자 계정/자격 증명/Google Cloud 프로젝트*: 계정 및 구독 세부 정보는 콘솔 에이전트를 기반으로 자동으로 채워집니다. 계정이나 구독이 여러 개인 경우 사용할 항목을 선택하십시오. 콘솔 에이전트를 사용할 수 없는 경우 https://docs.netapp.com/us-en/console-setup-admin/concept-agent-install-options.html["콘솔 에이전트 생성"^]하라는 메시지가 표시됩니다.
. *이름*: Cloud Volumes ONTAP 시스템(클러스터 이름이라고도 함)의 이름을 입력하십시오. NetApp Console은 시스템(클러스터) 이름을 사용하여 Cloud Volumes ONTAP 시스템과 VM 인스턴스의 이름을 지정합니다. 또한, 사전 정의된 보안 그룹을 선택한 경우 이 이름을 접두사로 사용합니다.
. *ONTAP 자격 증명*: Cloud Volumes ONTAP 클러스터 관리자 계정에 대한 자격 증명입니다. 이러한 자격 증명을 사용하여 ONTAP System Manager 또는 ONTAP CLI를 통해 Cloud Volumes ONTAP에 연결할 수 있습니다. 기본 _admin_ 사용자 이름을 유지하거나 사용자 지정 사용자 이름으로 변경할 수 있습니다.
. *태그/레이블*: 태그는 AWS 또는 Azure 리소스에 대한 메타데이터입니다. 콘솔은 Cloud Volumes ONTAP 인스턴스와 해당 인스턴스에 연결된 각 리소스에 태그를 추가합니다. Cloud Volumes ONTAP 시스템을 생성할 때 사용자 인터페이스에서 최대 15개의 태그를 추가할 수 있으며, 시스템 생성 후에도 태그를 추가할 수 있습니다. 자세한 내용은 다음을 참조하십시오.

.배포 및 구성
. *배포 유형/토폴로지*: 사용할 배포 유형을 선택합니다. 단일 노드, 단일 가용성 영역(AZ)의 고가용성(HA) 또는 여러 AZ의 HA를 선택할 수 있습니다.
. *네트워크 구성*: 지역, VNet 및 서브넷에 대한 계획 워크시트에 기록한 네트워크 정보를 입력하십시오.
+
.. *지역*: 기본적으로 리소스와 연결된 클라우드 계정의 지역이 선택됩니다.
.. *가용 영역*: 선택한 배포 유형에 따라 가용 영역을 선택할 수 있으며, 선택한 지역에서 가용 영역을 지원하는 경우에만 가능합니다.
.. *서브넷*: 지역에 사용할 서브넷을 선택합니다.
+
[role="tabbed-block"]

.AWS의 경우
--
*VPC*: AWS의 경우, 계획된 배포에 따라 서브넷이 있는 해당 리전의 Amazon Virtual Private Cloud(VPC) 네트워크를 입력합니다.

.AWS 고가용성
HA 구성을 선택한 경우 다음 정보를 입력하십시오.

.단일 AZ의 HA
... *중재자 접근*: 중재자 접근 정보를 지정합니다.  중재자는 HA 쌍의 상태를 모니터링하고 장애 발생 시 쿼럼을 제공하는 별도의 인스턴스입니다.  AWS EC2 서비스에 연결할 수 있도록 중재자 인스턴스에 키 쌍 이름을 제공하고 연결 방법을 선택합니다.


.여러 AZ의 HA
... *가용성 영역 및 중재자*: 각 노드와 중재자에 대한 가용성 영역(AZ)과 Cloud Volumes ONTAP HA 쌍을 배포하려는 해당 서브넷을 선택합니다.
... *유동 IP*: 여러 AZ를 선택한 경우 NFS 및 CIFS 서비스와 클러스터 및 SVM 관리를 위한 유동 IP 주소를 지정합니다.  IP 주소는 해당 지역의 모든 VPC에 대한 CIDR 블록 외부에 있어야 합니다.  추가 세부 사항은 다음을 참조하세요.link:https://docs.netapp.com/us-en/bluexp-cloud-volumes-ontap/reference-networking-aws.html#requirements-for-ha-pairs-in-multiple-azs["여러 AZ에서 Cloud Volumes ONTAP HA에 대한 AWS 네트워킹 요구 사항"^] .
... *중재자 접근*: 중재자 접근 정보를 지정합니다.  중재자는 HA 쌍의 상태를 모니터링하고 장애 발생 시 쿼럼을 제공하는 별도의 인스턴스입니다.  AWS EC2 서비스에 연결할 수 있도록 중재자 인스턴스에 키 쌍 이름을 제공하고 연결 방법을 선택합니다.
... *경로 테이블*: 여러 AZ를 선택한 경우, 플로팅 IP 주소에 대한 경로가 포함된 경로 테이블을 선택합니다.  두 개 이상의 경로 테이블이 있는 경우 올바른 경로 테이블을 선택하는 것이 중요합니다.  그렇지 않으면 일부 클라이언트가 Cloud Volumes ONTAP HA 쌍에 액세스하지 못할 수도 있습니다.  경로 테이블에 대한 자세한 내용은 다음을 참조하세요. http://docs.aws.amazon.com/AmazonVPC/latest/UserGuide/VPC_Route_Tables.html["AWS 문서: 라우팅 테이블"^] .


.충전 및 서비스

.Azure용
--
*VNet*: 필요한 Azure 가상 네트워크(VNet)를 선택합니다. VNet 요구 사항에 대한 자세한 내용은 link:reference-networking-azure.html#networking-for-azure-vnet-encryption["Azure VNet 암호화를 위한 네트워킹"]을 참조하십시오.

--
.Google 클라우드용
--
*VPC 및 서브넷*: Google Cloud의 경우, 배포 계획에 따라 해당 지역의 Google Virtual Private Cloud(VPC) 네트워크와 서브넷을 입력하십시오.

.단일 AZ 또는 다중 AZ의 HA
HA 구성을 선택한 경우 다음 정보를 입력하십시오. *노드 1, 2 및 미디에이터용 영역*: 두 노드와 미디에이터에 대한 영역을 추가합니다. *연결* HA 구성을 위해 서로 다른 VPC 4개와 각 VPC의 서브넷을 선택합니다. VPC-1, VPC-2 및 VPC-3에 대한 자세한 내용은 link:reference-networking-gcp.html#requirements-specific-to-single-node-systems["VPC-1, VPC-2 및 VPC-3에 대한 규칙"]을 참조하십시오.


NOTE: Cloud Volumes ONTAP 9.19.1 이상 버전에서는 link:concept-ha-google-cloud.html["단일 AZ HA 구성을 위한 공유 디스크"]을 지원하여 미디에이터 인스턴스를 선택 사항 또는 중복으로 만들 수 있습니다. 이 구성에서는 데이터 복제를 위한 미디에이터 영역, VPC 및 서브넷이 필요하지 않습니다.

.당신이 완료한 후
콘솔은 Cloud Volumes ONTAP 시스템을 배포합니다.  감사 페이지에서 진행 상황을 추적할 수 있습니다.

Cloud Volumes ONTAP 시스템 배포 중 문제가 발생하면 오류 메시지를 확인하십시오. 시스템에서 *환경 다시 생성*을 선택하여  `Advanced` 모드 마법사를 사용하여 배포를 계속 진행할 수도 있습니다.


CAUTION: 배포 프로세스가 완료된 후에는 AWS 또는 Azure 클라우드 포털에서 시스템에서 생성된 Cloud Volumes ONTAP 구성, 특히 시스템 태그를 수정하지 마십시오. 이러한 구성을 변경하면 예기치 않은 동작이나 데이터 손실이 발생할 수 있습니다.
