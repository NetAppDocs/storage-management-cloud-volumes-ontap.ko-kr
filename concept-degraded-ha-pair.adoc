= Cloud Volumes ONTAP HA 구성
:allow-uri-read: 


[role="lead"]
Cloud Volumes ONTAP의 고가용성(HA) 구성은 기업 스토리지에 대한 비즈니스 연속성과 내결함성을 보장합니다. 별도의 장애 도메인에 두 개의 ONTAP 노드를 배포함으로써 시스템은 하드웨어 장애, 네트워크 중단 또는 정기적인 유지보수 발생 시 무중단 운영(NDO) 및 원활한 페일오버를 제공합니다.

아키텍처에 다음이 필요한 경우 HA 쌍을 배포합니다.

* *미션 크리티컬 워크로드*: 다운타임을 허용할 수 없는 데이터베이스 및 엔터프라이즈 애플리케이션.
* *중단 없는 업그레이드*: 스토리지를 오프라인 상태로 전환하지 않고 ONTAP 버전을 업그레이드하거나 기본 클라우드 컴퓨팅 인스턴스를 변경하는 것입니다.
* *다중 영역 복원력*: 클라우드 지역 내 개별 가용성 영역 장애로부터 보호합니다.




[[how-ha-pairs-work]]
== HA 쌍의 작동 방식

HA 구성은 관리 관점에서 액티브-액티브 클러스터로 작동하며, 개별 데이터 애그리게이트는 특정 노드가 소유합니다.

* *동기식 복제*: 기본 노드의 NVRAM에 기록된 데이터는 클라이언트에 쓰기 확인을 보내기 전에 파트너 노드의 NVRAM에 동기적으로 미러링됩니다.
* *미디에이터 인스턴스 지원*: 타사 미디에이터 인스턴스가 두 노드의 상태를 모니터링합니다. 기본 노드가 응답하지 않게 되면 미디에이터가 스토리지 테이크오버를 조정합니다.
* *스토리지 인계*: 생존한 노드가 파트너의 스토리지 애그리게이트 및 네트워크 인터페이스의 소유권을 인계받아 몇 초 내에 데이터 제공을 재개합니다.




[[ha-configuration-requirements-and-limitations]]
== HA 구성 요구 사항 및 제한 사항

Cloud Volumes ONTAP에서 HA 쌍을 배포할 때 다음 요구 사항 및 제한 사항을 고려하십시오.

* *컴퓨팅 크기 조정*: HA 쌍의 두 노드는 정확히 동일한 클라우드 컴퓨팅 인스턴스 유형 또는 머신 형태를 사용해야 합니다.
* *프로토콜 지원*: Hyper-V 및 SQL Server 워크로드에 대해 SMB/CIFS 지속적 가용성(CA)이 지원되지만, 특정 ONTAP 구성 플래그가 필요합니다.
* *장애 조치 시간*: 스토리지 인계는 일반적으로 초 단위로 측정되지만, 애플리케이션 중단을 방지하려면 클라이언트 측 시간 초과 설정(예: iSCSI 이니시에이터 시간 초과)을 올바르게 구성해야 합니다.




[[operations-unavailable-when-a-node-in-an-ha-pair-is-offline]]
== HA 쌍의 노드가 오프라인 상태일 때 사용할 수 없는 작업

HA 쌍의 노드를 사용할 수 없는 경우 다른 노드가 파트너에게 데이터를 제공하여 지속적인 데이터 서비스를 제공합니다.  이것을 _스토리지 인수_라고 합니다.  보관된 물품을 반환하는 작업이 완료될 때까지 여러 작업을 수행할 수 없습니다.


TIP: HA 쌍의 노드를 사용할 수 없는 경우 NetApp Console 의 시스템 상태는 _저하_됩니다.

저장소 인수에서는 다음 작업을 수행할 수 없습니다.

* 지원 등록
* 라이센스 변경
* 인스턴스 또는 VM 유형 변경
* 쓰기 속도 변경
* CIFS 설정
* 구성 백업 위치 변경
* 클러스터 비밀번호 설정
* 디스크 및 집계 관리(고급 할당)


이러한 작업은 저장 공간 반환이 완료되고 시스템 상태가 정상으로 돌아온 후에 다시 사용할 수 있습니다.



[[storage-vm-disaster-recovery-for-cloud-volumes-ontap]]
== Cloud Volumes ONTAP용 스토리지 VM 재해 복구

NetApp Console은 스토리지 VM(SVM) 재해 복구(DR)에 대한 설정 또는 오케스트레이션 지원을 제공하지 않습니다. Cloud Volumes ONTAP의 DR을 설정하려면 ONTAP System Manager 또는 ONTAP CLI를 사용하여 SnapMirror SVM 복제를 구성해야 합니다.

두 Cloud Volumes ONTAP 시스템 간에 SnapMirror SVM 복제를 설정하는 경우, 복제는 두 개의 HA 쌍 시스템 또는 두 개의 단일 노드 시스템 간에만 가능합니다. HA 쌍 시스템과 단일 노드 시스템 간에는 SnapMirror SVM 복제를 설정할 수 없습니다.

ONTAP CLI 지침은 다음 문서를 참조하세요.

* https://library.netapp.com/ecm/ecm_get_file/ECMLP2839856["SVM 재해 복구 준비 익스프레스 가이드"^]
* https://library.netapp.com/ecm/ecm_get_file/ECMLP2839857["SVM 재해 복구 익스프레스 가이드"^]
