=== AWS Marketplace에서 Cloud Volumes ONTAP의 간편한 온보딩 소개

NetApp은 AWS Marketplace에서 Cloud Volumes ONTAP을 위한 간소화된 온보딩 워크플로를 도입하여 수동 작업과 온보딩 과정의 마찰을 줄였습니다.

이전에는 AWS Marketplace에서 Cloud Volumes ONTAP을 배포하려면 여러 화면을 탐색하고 역할, 정책 및 자격 증명을 수동으로 구성해야 했습니다. 이로 인해 온보딩 속도가 느리고 오류 발생 가능성이 높았습니다.

새로운 온보딩 워크플로는 다음과 같은 기능을 통해 마켓플레이스 구독에서 완벽하게 작동하는 Cloud Volumes ONTAP 환경으로의 전환을 간소화합니다.

. AWS Marketplace의 Cloud Volumes ONTAP 전용 상품 목록 구독: AWS Marketplace에서 Cloud Volumes ONTAP 독립형 상품 목록을 구독하고, 구독을 구매한 후, AWS 계정을 설정합니다(계정이 없는 경우).
. NetApp 콘솔의 안내 경로: NetApp 콘솔에서 계정을 설정하면 콘솔이 자동으로 배포 여정을 안내하며, 오른쪽 측면에 안내 경로를 표시합니다. 콘솔을 종료해야 하는 경우 나중에 안내 경로를 다시 시작할 수 있습니다.
. 구독 연동: 구독을 콘솔에 자동으로 연동하여 수동으로 연결하는 번거로움을 줄일 수 있습니다.
. AWS 자격 증명 구성을 사용한 Console 에이전트 배포: 필요한 AWS 역할을 자동으로 생성할 수 있습니다. 이 프로세스는 필요한 역할과 정책이 포함된 AWS CloudFormation 스택을 AWS 계정에 배포하여 검토할 수 있도록 한 다음, NetApp Console과 연결하여 수동 IAM 설정을 제거합니다. 그런 다음 에이전트 구성, 네트워킹 및 보안 그룹 설정을 진행하여 에이전트 배포를 시작할 수 있습니다.
. AWS에 새로운 Cloud Volumes ONTAP 시스템 배포:  `Quick create` 마법사로 바로 이동하여 더욱 빠르고 간편하게 Cloud Volumes ONTAP를 배포할 수 있습니다.

=== Azure의 Cloud Volumes ONTAP에 대한 잘 설계된 구성 분석

NetApp Console에서 이제 Microsoft Azure에서 실행되는 Cloud Volumes ONTAP 시스템에 대한 잘 설계된 구성 분석 기능을 지원합니다. Console은 배포 환경을 매일 분석하여 잘 설계된 상태, 인사이트 및 권장 사항을 제공함으로써 NetApp 모범 사례 및 잘 설계된 프레임워크에 부합하는 시스템을 유지 관리할 수 있도록 지원합니다.

잘 설계된 대시보드는 분석된 시스템의 총 개수를 표시하고 구성 및 심각도 수준별로 리소스를 분류합니다. 시스템 관리자인 경우 Console 대시보드에서 조직 내 구성 문제에 대한 권장 사항을 직접 확인하고 모든 권장 사항이 포함된 CSV 보고서를 다운로드할 수 있습니다. 구성은 다음 기준에 따라 분류됩니다.

* *안정성*: 장애 발생 시에도 워크로드가 정확하고 일관되게 작동하도록 보장합니다.
* *Security*: 위험 평가 및 완화를 통해 데이터, 시스템 및 자산을 보호합니다.
* *운영 우수성*: 비즈니스 가치를 극대화하기 위해 아키텍처를 최적화합니다.
* *비용 최적화*: 비용을 절감하고 최적화하여 더 큰 비즈니스 가치를 실현합니다.

[[support-for-next-generation-google-cloud-vms-in-cloud-volumes-ontap-ha]]
=== Cloud Volumes ONTAP HA 에서 차세대 Google Cloud VM 지원

9.19.1에서 NetApp은 Cloud Volumes ONTAP 고가용성(HA) 배포를 N2 VM에서 차세대 Google Cloud C3 시리즈 VM으로 전환하여 더욱 빠르고 확장 가능한 환경을 제공합니다.


NOTE: 이 지원은 단일 가용 영역(AZ) 고가용성(HA) 배포에만 적용됩니다.

이제 Google Cloud에서 Cloud Volumes ONTAP 9.19.1 이상 버전을 배포할 때 C3 시리즈 VM을 활용할 수 있습니다. 이러한 VM은 향상된 성능과 더 높은 용량 한도를 제공합니다. Google Virtual NIC(gVNIC)와 Hyperdisk Balanced 디스크를 사용하여 집약적인 워크로드에서 동적인 성능을 제공합니다.

Cloud Volumes ONTAP 시스템이 9.19.1 이상 버전인 경우, 단일 AZ에 간편하게 배포할 수 있도록 사전 구성된 패키지는 자동으로 C3 VM을 사용하며, 워크로드 요구 사항에 따라 IOPS 및 처리량 매개변수를 사용자 지정할 수 있습니다. 마찬가지로, 애그리게이트를 생성할 때 Google Cloud에서 성능과 확장성을 향상시키기 위해 Zonal Hyperdisk Balanced 디스크를 추가할 수 있습니다. 또한 기본 Flash Cache 지원을 위해 C3 시리즈 머신의 LSSD 버전을 선택할 수도 있습니다.

다음 사항에 유의하십시오:

* Cloud Volumes ONTAP 배포 환경에서는 C3 VM을 N2-series VM으로 또는 그 반대로 변경할 수 없습니다. 하지만 C3 제품군 내의 다른 변형 인스턴스 유형 간에는 전환이 가능합니다.
+

NOTE: C3 제품군 내에서 인스턴스 유형을 변경하는 데 걸리는 시간은 연결된 데이터 디스크 수에 따라 달라질 수 있습니다. 성능 저하를 방지하려면 사전에 구성을 계획하십시오.

* 인스턴스 유형을 변경할 때 대상 VM으로 c3-standard-44 또는 c3-standard-88과 해당 로컬 SSD(LSSD) 변형을 선택하면 Google 네트워크 서비스 송신 네트워크 계층이 자동으로 _Tier-1_로 적용됩니다. 다른 C3 변형으로 되돌리면 송신 네트워크 계층은 _Standard_로 설정됩니다.
+

NOTE: 송신 네트워크 계층을 _Tier-1_로 변경하면 추가 비용이 발생합니다. 배포 및 업그레이드 계획을 적절히 수립하십시오.

* C3의 LSSD 변형을 선택하면 Flash Cache가 기본적으로 활성화됩니다. 대상 변형이 Flash Cache를 지원하더라도 Cloud Volumes ONTAP 배포 환경에서는 C3 LSSD 변형 간에 전환할 수 없습니다.
* C3 VM에서 새 애그리게이트와 기존 애그리게이트 모두에 볼륨을 추가할 수 있습니다.
* N2 VM 유형의 Cloud Volumes ONTAP 시스템을 C3 VM 유형으로 복제할 때 디스크 유형은 기본적으로 Hyperdisk Balanced로 설정됩니다.
* NetApp Console 또는  `PUT /gcp/ha/aggregates/{workingEnvironmentId}/{aggregateName}` API를 사용하여 애그리게이트를 추가할 때 IOPS 및 처리량 설정을 변경할 수 있습니다.


https://docs.netapp.com/us-en/cloud-volumes-ontap-relnotes/reference-configs-gcp.html["Google Cloud에서 Cloud Volumes ONTAP에 대해 지원되는 구성"^]

[[introducing-access-roles-for-cloud-volumes-ontap]]
=== Cloud Volumes ONTAP의 액세스 역할 소개

NetApp은 Cloud Volumes ONTAP에 대한 ID 및 액세스 관리(IAM) 역할을 도입하여 역할 기반 액세스 제어(RBAC)를 통해 보안을 강화하고 규정 준수를 간소화합니다. 사용자는 NetApp Console에 기존 자격 증명을 사용하여 로그인하고 Cloud Volumes ONTAP 작업을 계속 수행할 수 있습니다. 그러나 이제 인증은 ID 페더레이션을 통해 이루어지므로 Console 관리자가 이러한 기능을 더욱 세밀하게 제어할 수 있습니다. 모든 사용자는 Cloud Volumes ONTAP에서 작업을 수행하려면 다음 역할 중 하나를 보유해야 합니다. 역할이 없으면 사용자는 Cloud Volumes ONTAP 인스턴스를 보거나 액세스할 수 없습니다.

* *Cloud Volumes ONTAP 뷰어*: Cloud Volumes ONTAP 화면 및 탭에 대한 읽기 전용 액세스 권한을 부여합니다. 이 역할에서는 특정 화면 및 작업 버튼이 비활성화됩니다.
* *Cloud Volumes ONTAP operator*: 볼륨 추가, 스토리지 관리, 복제 구성과 같은 일상적인 관리 작업을 지원합니다.
* *Cloud Volumes ONTAP admin*: 모든 Cloud Volumes ONTAP 작업에 대한 전체 액세스 권한을 제공합니다.


Console 관리자는 *Administration > Identity*에서 Cloud Volumes ONTAP 사용자에게 역할을 할당합니다. 역할 권한은 사용자의 소속 조직을 기준으로 하며 해당 조직 내의 모든 Cloud Volumes ONTAP 시스템에 적용됩니다.

Cloud Volumes ONTAP은 각 사용자의 권한을 로컬 캐시에 저장하여 RBAC를 적용하며, 이 캐시는 매시간 자동으로 갱신됩니다. 사용자에게 새 역할이 할당되거나 기존 역할이 변경되면 변경 사항이 적용되는 데 최대 1시간이 소요될 수 있으며, 이후 해당 역할에서 허용된 모든 워크플로가 활성화됩니다.


NOTE: 콘솔 에이전트를 처음 배포하거나 기존 콘솔 에이전트를 Console 4.9.0 이상 버전으로 업그레이드할 때, 일회성 시딩 프로세스가 조직의 모든 기존 사용자에게 Cloud Volumes ONTAP 역할을 자동으로 할당합니다. 이 프로세스는 완료하는 데 최대 30분이 소요될 수 있습니다. 완료되면 콘솔은 기존 사용자에게 적절한 역할을 할당합니다.

Cloud Volumes ONTAP 역할은 Amazon Web Service(AWS), Microsoft Azure(Azure) 및 Google Cloud 환경에 적용됩니다.

[[support-for-new-ec2-instances-for-cloud-volumes-ontap-in-aws]]
=== AWS의 Cloud Volumes ONTAP용 새 EC2 인스턴스 지원

Cloud Volumes ONTAP 9.16.1 이상 버전은 AWS에서 신규 및 기존 단일 노드와 고가용성(HA) 쌍 배포 모두에 대해 M6id 및 M6idn EC2 인스턴스 제품군을 지원합니다. 이러한 6세대 인스턴스는 5세대 범용 시스템에 비해 더 빠른 처리 속도, 강력한 네트워킹 기능 및 향상된 성능을 제공합니다.

* *M6id 인스턴스*: 고속 저지연 로컬 NVMe 스토리지를 통해 컴퓨팅 성능과 메모리 용량의 균형을 제공합니다. 모든 M6id 인스턴스 크기는 Flash Cache를 지원합니다. 이러한 인스턴스는 데이터 로깅 및 미디어 처리와 같은 워크로드에 적합합니다.
* *M6idn 인스턴스*: M6id 인스턴스와 동일한 로컬 NVMe 스토리지 기능을 제공합니다. 모든 M6idn 인스턴스 크기는 Flash Cache를 지원합니다.


고성능 또는 메모리 집약적인 워크로드에는 m6id.xlarge, m6id.2xlarge, m6idn.xlarge 및 m6idn.2xlarge 인스턴스 유형을 권장하지 않습니다. 이러한 워크로드에는 코어 개수와 메모리 용량이 더 큰 인스턴스 유형을 사용하십시오.

자세한 내용은 https://docs.netapp.com/us-en/cloud-volumes-ontap-relnotes/reference-configs-aws.html["AWS의 Cloud Volumes ONTAP 에 지원되는 구성"^]을 참조하세요.

[[flexible-billing-for-cloud-volumes-ontap-and-data-services-in-aws]]
=== AWS의 Cloud Volumes ONTAP 및 데이터 서비스에 대한 유연한 요금 청구

NetApp은 AWS에서 Cloud Volumes ONTAP 구독에 대한 향상된 청구 옵션을 제공하여 Cloud Volumes ONTAP 용량 및 NetApp Backup and Recovery와 같은 NetApp 데이터 서비스 비용 지불 방식을 더욱 세밀하게 제어할 수 있도록 지원합니다. 이전에는 모든 Cloud Volumes ONTAP 및 데이터 서비스 사용량이 AWS 마켓플레이스의 NetApp Intelligent Services(NIS) 목록을 통해 통합적으로 청구되었습니다.

AWS 엔터프라이즈 할인 프로그램(EDP) 변경으로 인해 NetApp은 AWS 마켓플레이스에 독립형 Cloud Volumes ONTAP 구독을 출시했습니다. 이를 통해 Cloud Volumes ONTAP 용량과 데이터 서비스 요금을 분리하여 청구할 수 있습니다. 이러한 분리를 통해 EDP 크레딧 및 비용 관리에 있어 더욱 유연성을 확보할 수 있습니다. NetApp은 Cloud Volumes ONTAP 사용량에 대한 비용 관리를 개선하기 위해 독립형 구독을 사용하는 것을 권장하며, EDP 요건이 적용되는 2026년 7월 31일 이후에는 데이터 서비스 요금 청구를 위해 NIS 구독을 계속 사용할 것을 권장합니다.

AWS의 독립형 Cloud Volumes ONTAP 구독은 유효한 용량 기반 Essentials 또는 Freemium 라이센스 패키지만 지원합니다. Professional 패키지는 지원되지 않습니다. 필요에 가장 적합한 청구 방법을 선택할 수 있습니다.

* *독립형 Cloud Volumes ONTAP 구독(권장)*: AWS 마켓플레이스에서 독립형 구독을 통해 Cloud Volumes ONTAP 사용량에 대한 비용을 지불하고, 데이터 서비스 요금은 NIS 구독을 통해 계속 청구됩니다. 이 옵션은 NetApp Console의 AWS 계정에 독립형 및 NIS 구독이 모두 연결된 대부분의 고객에게 권장됩니다.
* *통합 NIS 구독*: 기존 NIS 목록을 통해 Cloud Volumes ONTAP과 데이터 서비스를 함께 결제하세요. 이 통합 방식은 NetApp Console에서 AWS 계정에 NIS 구독만 연결된 기존 고객에게 적용됩니다. 하지만 NetApp은 더 나은 비용 관리를 위해 Cloud Volumes ONTAP 사용량에 대한 결제 기본 설정을 지정하고 독립형 구독으로 전환하는 것을 권장합니다.


구독 유형에 따라 다음과 같은 시나리오가 제공됩니다.

.독립형 구독만 가능(신규 고객)
NetApp Console의 AWS 계정에 독립형 AWS 구독만 연결되어 있고 NIS 구독이 없는 경우, 배포 시 Essentials 라이선스 패키지만 선택할 수 있습니다. 데이터 서비스 *Backup and Recovery* 옵션을 사용할 수 없습니다.

.독립 실행형 및 NIS 구독(기존 고객) 모두
AWS 계정에 NIS 구독과 독립 실행형 구독이 모두 연결되어 있고 NetApp Console의 *청구 기본 설정* 탭에서 청구 기본 설정을 지정하지 않은 경우 Cloud Volumes ONTAP 사용량은 기본적으로 NIS를 통해 청구됩니다. 독립 실행형 목록에서는 Professional을 지원하지 않으므로 Professional 패키지를 실행하는 모든 Cloud Volumes ONTAP 인스턴스에서도 기본적으로 이 방식이 적용됩니다. 청구 기본 설정을 지정하여 각 서비스에 대한 청구 방식을 선택할 수 있습니다.

* 독립형 구독을 Cloud Volumes ONTAP(Essentials 패키지 포함)의 기본 결제 계정으로 설정하세요. Professional 패키지만 있는 경우 Essentials 패키지로 전환하고 결제 기본 설정을 지정하여 Cloud Volumes ONTAP 사용량을 청구할 수 있습니다. 이 방법은 Cloud Volumes ONTAP EDP 크레딧을 보다 효과적으로 관리하는 데 도움이 되므로 권장합니다.
* NIS 구독을 사용하여 데이터 서비스 요금을 청구하거나 Professional 패키지를 통해 Cloud Volumes ONTAP 사용량을 청구합니다. 배포 중에 *Backup and Recovery* 데이터 서비스를 활성화하고 Professional 라이센스 패키지를 선택하여 NIS 구독을 통해 데이터 서비스 요금을 청구합니다.


.NIS 구독 전용(기존 고객)
이는 AWS 계정과 연결된 NIS 구독을 보유한 기존 고객에게 적용됩니다. 현재와 같이 Cloud Volumes ONTAP과 데이터 서비스를 함께 청구할 수 있습니다. 하지만 NetApp은 Cloud Volumes ONTAP 사용량에 대한 비용 관리를 개선하기 위해 독립형 구독 방식으로 전환하는 것을 권장합니다.

* AWS 포털에서 새로운 독립형 목록을 구독하세요.
* 독립형 구독을 AWS 계정에 연결하세요.
* Essentials 패키지를 사용한 Cloud Volumes ONTAP 사용에 대해 결제 기본 설정을 독립형 구독으로 설정하십시오.
* (해당되는 경우) Professional 패키지를 Essentials 패키지로 변경하고 결제 방식을 설정하세요.
* NIS 구독을 통해 데이터 서비스 요금 청구를 계속 진행하십시오.



NOTE: 여러 구독 유형을 사용하거나 목록 간에 마이그레이션하는 경우 중복 측정이 발생하지 않습니다.

[[improved-cloud-volumes-ontap-support-with-google-cloud-c3-vms]]
=== Google Cloud C3 VM에서 Cloud Volumes ONTAP 지원이 향상되었습니다.

Cloud Volumes ONTAP는 이제 Google Cloud C3 시리즈 VM에 대한 지원을 강화하여 다음과 같은 추가 기능을 사용할 수 있게 되었습니다.

* Cloud Volumes ONTAP 배포에서 C3 제품군 내의 다양한 변형 간에 인스턴스 유형을 전환할 수 있습니다.
* 인스턴스 유형을 변경할 때 대상 VM으로 c3-standard-44 또는 c3-standard-88 및 해당 LSSD 변형을 선택하면 Google 네트워크 서비스 송신 네트워크 계층 Tier-1이 자동으로 적용됩니다. 다른 C3 변형으로 전환하면 송신 네트워크 계층은 Standard로 설정됩니다.
+

NOTE: 송신 네트워크 계층을 Tier-1으로 변경하면 추가 비용이 발생합니다. 배포 및 업그레이드 계획을 적절히 수립하십시오.

* C3의 로컬 SSD(LSSD) 변형을 선택하면 Flash Cache가 기본적으로 활성화됩니다. 대상 변형이 Flash Cache를 지원하더라도 Cloud Volumes ONTAP 배포 환경에서는 C3 LSSD 변형 간에 전환할 수 없습니다.
* 이제 C3 VM에서 새 애그리게이트와 기존 애그리게이트 모두에 볼륨을 추가할 수 있습니다.
* NetApp Console을 통해 또는  `PUT /gcp/vsa/aggregates/{workingEnvironmentId}/{aggregateName}` API를 사용하여 볼륨을 추가할 때 IOPS 및 처리량 설정을 변경할 수 있습니다.

`PUT /occm/config`

요청 본문에 다음 매개변수를 포함하십시오.

[source, json]

{

}
----
기본 모드는 표준 모드입니다. 즉, Console 에이전트가 20분 동안 Cloud Volumes ONTAP에 연결할 수 없는 경우 프록시 설정을 기본값으로 덮어씁니다.

[[cloud-volumes-ontap-support-for-next-generation-google-cloud-vms]]
=== 차세대 Google Cloud VM을 위한 Cloud Volumes ONTAP 지원

NetApp은 9.18.1부터 새로운 Cloud Volumes ONTAP 배포를 N2 VM에서 차세대 Google Cloud C3 시리즈 VM으로 전환하여 더욱 빠르고 확장 가능한 환경을 제공합니다. 이제 Google Cloud에서 Cloud Volumes ONTAP 9.18.1 이상 버전을 배포할 때 C3 시리즈 VM의 이점을 활용할 수 있습니다. C3 시리즈 VM은 Google Virtual NIC(gVNIC)와 Hyperdisk Balanced 디스크를 사용하여 향상된 성능과 더 높은 용량 한도를 제공하며, 집약적인 워크로드에 필요한 동적 성능을 보장합니다.


NOTE: 현재 Cloud Volumes ONTAP는 단일 노드 배포 환경에서만 C3 시리즈를 지원합니다.

Cloud Volumes ONTAP 시스템이 9.18.1 이상을 실행하는 경우 간편한 단일 노드 배포에 사용하는 사전 구성된 패키지는 자동으로 C3 VM을 사용하며 워크로드 요구 사항에 따라 IOPS 및 처리량 매개 변수를 사용자 지정할 수 있습니다. 마찬가지로 애그리게이트를 생성하는 동안 Hyperdisk Balanced 디스크를 추가하여 Google Cloud에서 더 나은 성능과 확장성을 달성할 수 있습니다. 또한 기본 Flash Cache 지원을 위해 C3 시리즈 머신의 LSSD 변형을 선택할 수 있습니다.

C3 VM은 Hyperdisk Balanced 디스크만 지원하기 때문에 애그리게이트에 볼륨을 추가할 때 디스크 유형을 변경할 수 없습니다. 마찬가지로 N2 VM 유형의 시스템을 C3 VM으로 복제할 때 디스크 유형은 기본적으로 Hyperdisk Balanced로 설정됩니다.

[[restricted-availability-of-byol-licensing-for-cloud-volumes-ontap]]
=== Cloud Volumes ONTAP 에 대한 BYOL 라이선싱의 제한된 가용성

2025년 6월 25일부터 NetApp Cloud Volumes ONTAP 의 BYOL(Bring Your Own License) 라이선스 모델을 제한했습니다. 이 제한은 AWS, Azure, Google Cloud의 모든 고객 및 Cloud Volumes ONTAP 배포에 적용됩니다. 미국 공공 부문 고객과 중국 리전 배포는 예외입니다.

NetApp 지원 및 서비스는 BYOL 계약이 만료될 때까지 계속되지만, 만료된 라이선스는 갱신 또는 연장되지 않습니다. BYOL 라이선스가 만료되면 클라우드 마켓플레이스 구독을 통해 구매한 용량 기반 라이선스로 교체해야 합니다. 하이퍼스케일러 마켓플레이스를 통한 용량 기반 라이선스 모델은 라이선스 경험을 간소화하고 더 큰 비즈니스 이점을 제공합니다. 전환 옵션에 대해 논의하려면 NetApp 계정 팀 또는 고객 성공 담당자에게 문의하십시오.

자세한 내용은 다음 고객 공지문을 참조하세요.  https://mysupport.netapp.com/info/communications/CPC-00661.html["CPC-00661: Cloud Volumes ONTAP BYOL 정책 변경"^] .

NetApp Cloud Volumes ONTAP 노드 기반 라이선싱의 가용성 종료(EOA) 및 지원 종료(EOS)를 계획했습니다.  2024년 11월 11일부터 노드 기반 라이선스의 제한된 제공이 종료되었습니다. 노드 기반 라이선싱에 대한 지원은 2024년 12월 31일에 종료됩니다.  노드 기반 라이선스의 EOA가 만료된 후에는 BlueXP 라이선스 변환 도구를 사용하여 용량 기반 라이선스로 전환해야 합니다.

연간 또는 장기 약정의 경우 NetApp EOA 날짜 또는 라이선스 만료일 전에 NetApp 담당자에게 연락하여 전환에 필요한 전제 조건이 충족되었는지 확인할 것을 권장합니다.  Cloud Volumes ONTAP 노드에 대한 장기 계약이 없고 온디맨드 PAYGO(Pay-as-you-go) 구독으로 시스템을 실행하는 경우 EOS 날짜 전에 전환을 계획하는 것이 중요합니다.  장기 계약과 PAYGO 구독 모두 BlueXP 라이선스 변환 도구를 사용하여 원활하게 변환할 수 있습니다.

https://docs.netapp.com/us-en/bluexp-cloud-volumes-ontap/concept-licensing.html#end-of-availability-of-node-based-licenses["노드 기반 라이선스 제공 종료"^] https://docs.netapp.com/us-en/bluexp-cloud-volumes-ontap/task-convert-node-capacity.html["Cloud Volumes ONTAP 노드 기반 라이선스를 용량 기반 라이선스로 변환"^]

NetApp 계약 의무와 운영상의 필요 사항을 준수하기 위해 다음과 같은 고유한 라이선스 요구 사항을 인식하고 있으며, 이러한 상황에서 노드 기반 라이선스를 계속 지원할 것입니다.

* 미국 공공 부문 고객
* 개인 모드 배포
* AWS에서 Cloud Volumes ONTAP 의 중국 지역 배포
* 유효하고 만료되지 않은 노드별 BYOL 라이선스가 있는 경우

[[worm-and-arp-functionalities-are-no-longer-chargeable]]
=== WORM 및 ARP 기능은 더 이상 유료화되지 않습니다.

WORM(Write Once Read Many) 및 ARP(Autonomous Ransomware Protection)의 내장된 데이터 보호 및 보안 기능은 추가 비용 없이 Cloud Volumes ONTAP 라이선스와 함께 제공됩니다.  새로운 가격 모델은 AWS, Azure, Google Cloud의 새 BYOL 및 PAYGO/마켓플레이스 구독과 기존 구독 모두에 적용됩니다.  용량 기반 라이선스와 노드 기반 라이선스 모두 단일 노드와 고가용성(HA) 쌍을 포함한 모든 구성에 대한 ARP와 WORM을 추가 비용 없이 포함합니다.

간소화된 가격 책정으로 다음과 같은 혜택을 누리실 수 있습니다.

* 현재 WORM 및 ARP가 포함된 계정에는 이러한 기능에 대한 요금이 더 이상 부과되지 않습니다.  앞으로는 이 변경 전과 마찬가지로 용량 사용에 대해서만 요금이 청구됩니다.  WORM과 ARP는 더 이상 향후 청구서에 포함되지 않습니다.
* 현재 계정에 이러한 기능이 포함되어 있지 않은 경우 이제 추가 비용 없이 WORM 및 ARP를 선택할 수 있습니다.
* 모든 신규 계정에 대한 Cloud Volumes ONTAP 서비스에는 WORM 및 ARP 비용이 포함되지 않습니다.


이러한 기능에 대해 자세히 알아보세요.

* https://docs.netapp.com/us-en/bluexp-cloud-volumes-ontap/task-protecting-ransomware.html["Cloud Volumes ONTAP 에 NetApp 랜섬웨어 보호 솔루션 활성화"]
* https://docs.netapp.com/us-en/bluexp-cloud-volumes-ontap/concept-worm.html["WORM 스토리지"]

* Essentials 패키지와 Professional 패키지는 이제 모든 클라우드 공급업체 마켓플레이스에서 이용할 수 있습니다.
+

+
신규 고객인 경우, 새로운 작업 환경을 만들 때 BlueXP 구독하라는 메시지가 표시됩니다.

+
이 청구 방법은 활성 구독이 있는 기존 고객에게는 계속 제공됩니다.



https://docs.netapp.com/us-en/bluexp-cloud-volumes-ontap/concept-licensing.html["Cloud Volumes ONTAP 의 라이선싱 옵션에 대해 자세히 알아보세요."] .

=== AWS의 EBS 볼륨에 대한 태그

이제 BlueXP 새로운 Cloud Volumes ONTAP 작업 환경을 생성할 때 EBS 볼륨에 태그를 추가합니다.  태그는 Cloud Volumes ONTAP 배포된 후에 생성되었습니다.

조직에서 SCP(서비스 제어 정책)를 사용하여 권한을 관리하는 경우 이러한 변경 사항이 도움이 될 수 있습니다.



[[minimum-cooling-period-for-auto-tiering-policy]]
=== 자동 티어링 정책에 대한 최소 냉각 기간

_자동_ 계층화 정책을 사용하여 볼륨에서 데이터 계층화를 활성화한 경우 이제 API를 사용하여 최소 냉각 기간을 조정할 수 있습니다.

https://docs.netapp.com/us-en/bluexp-cloud-volumes-ontap/task-tiering.html#changing-the-cooling-period-for-the-auto-tiering-policy["최소 냉각 기간을 조정하는 방법을 알아보세요."]



[[enhancement-to-custom-export-policies]]
=== 사용자 정의 수출 정책 향상

새로운 NFS 볼륨을 생성할 때 BlueXP 이제 사용자 정의 내보내기 정책을 오름차순으로 표시하여 필요한 내보내기 정책을 더 쉽게 찾을 수 있게 되었습니다.



[[deletion-of-old-cloud-snapshots]]
=== 이전 클라우드 스냅샷 삭제

이제 BlueXP Cloud Volumes ONTAP 시스템이 배포될 때와 전원이 꺼질 때마다 생성되는 루트 및 부팅 디스크의 이전 클라우드 스냅샷을 삭제합니다.  루트 볼륨과 부트 볼륨 모두에 대해 가장 최근의 스냅샷 두 개만 보존됩니다.

이 향상된 기능은 더 이상 필요하지 않은 스냅샷을 제거하여 클라우드 공급자 비용을 줄이는 데 도움이 됩니다.

커넥터에는 Azure 스냅샷을 삭제하기 위한 새로운 권한이 필요합니다. https://docs.netapp.com/us-en/bluexp-setup-admin/reference-permissions-azure.html["Azure에 대한 최신 커넥터 정책 보기"^] .

[source, json]
----
"Microsoft.Compute/snapshots/delete"
----

[[logical-space-reporting]]
=== 논리적 공간 보고

이제 BlueXP Cloud Volumes ONTAP 에 대해 생성하는 초기 스토리지 VM에 대한 논리적 공간 보고를 활성화합니다.

공간이 논리적으로 보고되는 경우 ONTAP 저장 효율성 기능으로 절약된 모든 물리적 공간도 사용된 것으로 보고되도록 볼륨 공간을 보고합니다.



[[support-for-gp3-disks-in-aws]]
=== AWS에서 gp3 디스크 지원

Cloud Volumes ONTAP 이제 9.7 릴리스부터 _일반 용도 SSD(gp3)_ 디스크를 지원합니다. gp3 디스크는 광범위한 작업 부하에 대해 비용과 성능의 균형을 맞춘 가장 저렴한 SSD입니다.

https://docs.netapp.com/us-en/bluexp-cloud-volumes-ontap/task-planning-your-config.html["AWS에서 시스템 크기 조정"] .



[[cold-hdd-disks-no-longer-supported-in-aws]]
=== AWS에서는 콜드 HDD 디스크가 더 이상 지원되지 않습니다.

Cloud Volumes ONTAP 더 이상 Cold HDD(sc1) 디스크를 지원하지 않습니다.



[[tls-1-2-for-azure-storage-accounts]]
=== Azure 스토리지 계정용 TLS 1.2

BlueXP Azure에서 Cloud Volumes ONTAP 용 스토리지 계정을 생성할 때 스토리지 계정의 TLS 버전은 이제 1.2입니다.

[[aws-encryption-with-customer-managed-cmks]]
=== 고객 관리 CMK를 사용한 AWS 암호화

BlueXP 사용하면 AWS Key Management Service(KMS)를 사용하여 Cloud Volumes ONTAP 데이터를 암호화할 수 있습니다.  Cloud Volumes ONTAP 9.9.0부터 고객 관리 CMK를 선택하면 EBS 디스크의 데이터와 S3에 계층화된 데이터가 암호화됩니다.  이전에는 EBS 데이터만 암호화되었습니다.

CMK를 사용하려면 Cloud Volumes ONTAP IAM 역할에 액세스 권한을 제공해야 합니다.

https://docs.netapp.com/us-en/bluexp-cloud-volumes-ontap/task-setting-up-kms.html["Cloud Volumes ONTAP 사용하여 AWS KMS를 설정하는 방법에 대해 자세히 알아보세요."] .

[[ip-address-reduction-in-google-cloud]]

https://docs.netapp.com/us-en/bluexp-cloud-volumes-ontap/reference-networking-gcp.html#requirements-for-cloud-volumes-ontap["Google Cloud의 IP 주소 요구 사항에 대해 자세히 알아보세요."] .

[[aws-outposts]]
=== AWS 아웃포스트

몇 달 전, 우리는 Cloud Volumes ONTAP 이 Amazon Web Services(AWS) Outposts Ready 지정을 달성했다고 발표했습니다.  오늘, AWS Outposts에서 BlueXP 와 Cloud Volumes ONTAP 유효성을 검증했다는 소식을 전해드리게 되어 기쁩니다.

AWS Outpost가 있는 경우 작업 환경 마법사에서 Outpost VPC를 선택하여 해당 Outpost에 Cloud Volumes ONTAP 배포할 수 있습니다. 경험은 AWS에 있는 다른 VPC와 동일합니다. 먼저 AWS Outpost에 커넥터를 배포해야 합니다.

지적해야 할 몇 가지 제한 사항이 있습니다.

* 현재 단일 노드 Cloud Volumes ONTAP 시스템만 지원됩니다.
* Cloud Volumes ONTAP 과 함께 사용할 수 있는 EC2 인스턴스는 Outpost에서 사용 가능한 인스턴스로 제한됩니다.
* 현재는 일반용 SSD(gp2)만 지원됩니다.




[[ultra-ssd-vnvram-in-supported-azure-regions]]
=== 지원되는 Azure 지역의 Ultra SSD VNVRAM

Cloud Volumes ONTAP는 이제 단일 노드 시스템과 함께 E32s_v3 VM 유형을 사용할 때 Ultra SSD를 VNVRAM으로 사용할 수 있습니다 https://docs.microsoft.com/en-us/azure/virtual-machines/disks-enable-ultra-ssd["지원되는 모든 Azure 지역에서"^].

VNVRAM은 더 나은 쓰기 성능을 제공합니다.



[[choose-an-availability-zone-in-azure]]
=== Azure에서 가용성 영역 선택

이제 단일 노드 Cloud Volumes ONTAP 시스템을 배포할 가용성 영역을 선택할 수 있습니다.  AZ를 선택하지 않으면 BlueXP 대신 AZ를 선택해 드립니다.

image:screenshot_azure_az.gif["지역을 선택한 후 사용할 수 있는 가용성 영역 드롭다운 목록의 스크린샷입니다."]



[[larger-disks-in-google-cloud]]
=== Google Cloud의 더 큰 디스크

Cloud Volumes ONTAP는 이제 Google Cloud에서 64TB 디스크를 지원합니다.


NOTE: Google Cloud 제한으로 인해 디스크만으로 구성된 최대 시스템 용량은 256TB입니다.



[[new-machine-types-in-google-cloud]]
=== Google Cloud의 새로운 머신 유형

이제 Cloud Volumes ONTAP은 Google Cloud에서 다음과 같은 PAYGO 라이센스를 사용하여 다음 머신 유형을 지원합니다.

* Explore 라이선스와 BYOL을 사용하는 n2-standard-4
* Standard 라이선스와 BYOL을 갖춘 n2-standard-8
* 프리미엄 라이선스와 BYOL을 갖춘 n2-standard-32
